#+END_SRC

  - [-] errors as values
    - it's similar to Zig, in more ways than one
    - essentially a glorified enum
    - [-] comparison is identical to comparing an enum
      - the backing integers never collide, so this is safe (unless they share the same name)
    - [ ] errors can be grouped into a type (just called an error group)
      - /all/ errors are placed into a global group (simply named =error=)
      - behaves like a tagged union; but minus the runtime checks
      - groups are like (/untagged/) unions
      - [ ] error groups (intentionally) have no type safety
        - since errors of the same name have the same backing integer value, the following evaluates to true
#+BEGIN_SRC
          typedef error {
            OUT_OF_MEMORY,
            SOME_OTHER_ERROR,
          } FooError;
          typedef error {
            OUT_OF_MEMORY,
            SOME_OTHER_MORE_DIFFERENT_ERROR,
          } BarError;
          FooError me_err = .OUT_OF_MEMORY;
          BarError other_err = .OUT_OF_MEMORY
          no_op(me_err == other_err && me_err == error.OUT_OF_MEMORY && FooError.OUT_OF_MEMORY == error.OUT_OF_MEMORY);
#+END_SRC
        - however, you can enforce strict comparision (ie: group + value) (by explicitly stating it)
          using the same =typedef=s from above, this now evaluates to false
#+BEGIN_SRC
            FooError me_err = .OUT_OF_MEMORY;
            no_op(me_err == BarError.OUT_OF_MEMORY);
#+END_SRC

  - [ ] error unions (just a pre-defined tagged union)
    - this is almost identical to Zig's error unions
    - it is effectively just this typedef (as plain C) (assuming the function returns an =int= as the non-error state):
#+BEGIN_SRC
      typedef enum {
        INVALID_INPUT,
        CONNECTION_REFUSED,
        BAD_RESPONSE,
      } SomeError;
      typedef struct {
        enum { ok, err } tag;
        union {
          int ok;
          SomeError err;
        } value;
      } ErrorUnion;
#+END_SRC
    - this doesn't really need any explaination, but here I am
    - the usual =try= keyword is used to bubble-up an error
      - on error, this immediately returns from the function (with the error) (again, as usual)
    - errors are "caught" using the =onerr= keyword. Also, some side-notes:
      - I never understood why you'd =catch= it if you're not also =throw=-ing it
      - =catch=-ing the error sounds _very_ object oriented
    - if neither =try= nor =onerr= is used on a function which returns an error union, the type of the resulting value is the error union
      - a function returning =SomeError!char*= cannot be directly casted to =char*=
        - you /must/ either access a field in the error union (which's runtime checked in safe modes), or handle it
    - the error value is accessible under the =.err= field and the non-error value is accessible under the =.ok= field
      - just like a normal tagged union, comparison can be done on the active tag (if you don't want to use =try= or =onerr=); ie:
#+BEGIN_SRC
        MeError!char* alloc(int len) {
          char* str = malloc(len);
          if (str) return str;
          return error.OUT_OF_MEMORY;
        }
        int main(void) {
          MeError!char* me_str = alloc(4);
          assert(me_str != .err, "allocation failed");
          memcpy(me_str.ok, "foo", 4);
          puts(me_str.ok);
          free(me_str.ok);
          return 0;
        }
#+END_SRC
